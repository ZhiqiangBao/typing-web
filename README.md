# 打字练习 Typing Web

一个基于原生 HTML / CSS / JavaScript 的打字练习 Web 应用,内置训练档案统计与图表可视化,所有数据保存在本地浏览器中,无需登录、无需后端数据库。

## 功能特性

### 练习模式(Practice)
- 内置多类文章语料:洛丽塔、追忆似水年华、百年孤独、唐诗、英文谚语、Python/SQL/C++ 代码片段等
- 实时显示:进度、准确率、WPM、错误字符数
- 字符级高亮:未输入=灰色 · 正确=黑色 · 错误=红色 · 当前位置=蓝色下划线
- 支持随机切换文章 / 下拉选择指定文章 / 重置当前进度

### 训练档案(Profile)
- **总览统计**:总训练次数、总正确率、历史最高 WPM、平均 WPM
- **今日统计**:今日训练次数、今日正确率、今日最高 WPM、今日平均 WPM
- **WPM 趋势图**:折线图展示最近 30 次练习的 WPM 变化
- **错误字符数柱状图**:柱状图展示最近 30 次练习的错误字符数
- **历史记录列表**:最近 30 条练习详情(时间、文章、WPM、准确率、错误数)
- 一键清空历史(带确认对话框)

### 主题切换
内置 4 套主题,实时切换并持久化保存:
- 浅蓝(默认)
- 深色
- 米黄
- 青绿

## 快速开始

### 环境要求
- Python 3.7+(仅用于启动本地静态服务器)
- 现代浏览器(Chrome / Edge / Firefox / Safari)

### 启动

```bash
python server.py
```

启动后控制台会输出:

```
Typing app running at http://127.0.0.1:8000/  (Ctrl+C to stop)
```

浏览器打开 **http://127.0.0.1:8000/** 即可使用。

按 `Ctrl + C` 可停止服务。


### 添加自定义文章

把任意 `.txt` 或 `.md` 文件放入 `docs/` 目录,刷新页面即可在文章下拉框看到。

`app.js` 顶部的 `categorize()` 函数会根据文件名前缀自动分组,例如:
- `xxx_en.txt` → 标记为「英文」
- `xxx_zh.txt` → 标记为「中文」
- `code_xxx` / `python_code` / `sql_query` 等 → 归入「代码 / 符号」
- 其他 → 自动归入「其他」分组

## 数据存储

所有训练记录与主题偏好保存在浏览器的 `localStorage` 中:

| 键名 | 内容 | 格式 |
|---|---|---|
| `typing_history` | 历史训练记录数组 | JSON |
| `typing_theme` | 当前主题 | `blue` / `dark` / `beige` / `green` |

每条历史记录字段:

```json
{
  "ts": 1718000000000,        // 时间戳
  "day": "2026-07-14",        // 当日日期(本地时区)
  "doc": "lolita_en.txt",     // 文章文件名
  "wpm": 65,                  // 本次 WPM
  "acc": 98,                  // 本次准确率(0-100)
  "errors": 3,                // 错误字符数
  "typed": 412,               // 输入总字符数
  "correct": 409               // 正确字符数
}
```

> **注意**:数据与浏览器/域名绑定。清空浏览器缓存、更换浏览器或在隐私模式下打开,历史记录将不会保留。如需跨设备同步,需要自行扩展后端 API。

## 统计口径说明

- **总训练次数**:已保存的历史记录总条数
- **总正确率**:`Σ 各次正确字符 / Σ 各次输入字符 × 100`(按字符加权,比简单平均更准确)
- **最高 WPM**:所有记录中的最大值
- **平均 WPM**:`Σ 各次 WPM / 记录数`
- **当日统计**:仅统计 `day` 字段等于今日的记录

## 技术栈

- **前端**:原生 HTML5 / CSS3 / JavaScript(无构建步骤,无框架)
- **图表**:Chart.js 4.4.1(通过 CDN 引入)
- **后端**:Python `http.server`(标准库,零依赖)

## 常见问题

### 打开页面显示「无法连接服务器」

确认 `server.py` 正在运行。若端口 8000 被占用,修改 `server.py` 中的 `PORT` 常量。

### 切换主题后图表颜色没更新

主题切换时会自动重绘 Profile 图表。若 Profile 视图未打开,切过去时会重新渲染。

### 完成练习后 Profile 没有数据

- 确认输入框已打满整篇文章(进度 100%)
- 检查浏览器是否禁用了 `localStorage`(隐私模式下可能受限)
